# 模版：会议纪要（meeting_minutes）

本模版用于 Organizer 插件预处理：当 50_Raw 中的条目被识别为「会议纪要」类时，按以下结构从原文抽取并填充字段，再继续分类归位。

## 目标结构

AI 从原文中抽取信息，按以下格式整理（可选用 YAML frontmatter + Markdown 正文，或纯 Markdown 小标题）。

### 字段说明

| 字段 | 说明 | 可留空 |
|------|------|--------|
| 会议主题 | 会议名称或主题 | 否，尽量从原文提取 |
| 时间 | 会议时间（日期、时段） | 是，缺失时填「待补」 |
| 参会人 | 参会人员或部门 | 是，缺失时填「待补」 |
| 议题 | 讨论的议题列表或要点 | 是，无则留空 |
| 决议/结论 | 达成的结论、决定 | 是，无则留空 |
| 待办事项 | 后续待办（责任人、截止等） | 是，无则留空 |

### 输出格式示例（Markdown）

```markdown
---
type: meeting_minutes
会议主题: 
时间: 
参会人: 
议题: 
决议: 
待办: 
---

# 会议纪要

## 会议主题
（从原文抽取或概括）

## 时间
（会议日期、时段）

## 参会人
（参会人员或部门）

## 议题
（讨论的议题或要点列表）

## 决议/结论
（达成的结论、决定）

## 待办事项
（后续待办，含责任人、截止时间等；无则写「无」）

## 原始内容
（可选：保留一段原文便于追溯）
```

## 抽取指引

- **会议主题**：原文中的「会议名称」「主题」「关于 XXX 的会议」等。
- **时间**：原文中的日期、时间、「何时召开」「会议时间」等。
- **参会人**：原文中的「参会人」「出席」「与会人员」「部门」等。
- **议题**：讨论的议题、议程项、主要讨论点，可列为要点。
- **决议/结论**：达成的结论、决定、共识、结论摘要等。
- **待办事项**：后续行动项、责任人、截止时间；无则填「无」。
- 若某字段原文中完全无信息，填「待补」，不编造内容。
- 整理后保留与原文一致的语义，不改变事实性描述。
